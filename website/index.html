The Big Picture

Before diving into code, let's understand what GPT actually does. It's simpler than you think!

💡 GPT in One Sentence

GPT reads text one character at a time, learns the patterns, and predicts what comes next.

That's it! All the fancy math (attention, embeddings, transformers) is just different ways of making that prediction more accurate.

📝 Input: "ROMEO: O, she doth tea"
The raw text you feed to the model. It sees a sequence of characters.
↓
🔢 Tokenize → Numbers
Convert each character to a number. 'R'→44, 'O'→41, etc. Neural nets only understand numbers!
↓
📊 Embed → Vectors
Each number becomes a rich vector (list of numbers). These vectors capture meaning that the model learns during training.
↓
× N layers
👀 Attention — "Which past chars matter?"
The model looks back at all previous characters and figures out which ones are most relevant for predicting the next one.
↓
⚙️ Feed-Forward — "Process the info"
A small neural network that processes the combined information. Think of it as the model "thinking" about what it just read.
↓
🎯 Predict → next character = "c"
The model outputs a probability for each possible next character. "c" gets the highest probability (to complete "teach").

Tokenization

Neural networks can't read text — they only understand numbers. Tokenization converts every character into a unique number.

🔤 How It Works

  • Find all unique characters in the Shakespeare text (a, b, c, ..., A, B, C, ..., !, ?, newline, space, etc.)
  • Sort them and assign each a unique ID (0, 1, 2, ...)
  • encode("Hi") converts "Hi" → [20, 47]
  • decode([20, 47]) converts [20, 47] → "Hi"
step1_tokenize.py
1# Read Shakespeare text

5chars = sorted(set(text))
6vocab_size = len(chars)
7ch2i = {c: i for i, c in enumerate(chars)} # char → number
8i2ch = {i: c for i, c in enumerate(chars)} # number → char
9
10encode = lambda s: [ch2i[c] for c in s]
11decode = lambda l: ''.join(i2ch[i] for i in l)

📖 Line-by-Line Breakdown

  • Line 2 — Read the entire Shakespeare file into one long string
  • Line 5 — set(text) gets unique chars, sorted() puts them in order. Example: [' ', '!', 'A', 'B', ..., 'a', 'b', ...]
  • Line 6 — vocab_size = total unique characters (65 in Shakespeare). This is our "vocabulary"
  • Lines 7-8 — Two dictionaries: one maps char→number, the other maps number→char. These are inverse lookups
  • Lines 10-11 — encode: turns a string into a list of numbers. decode: reverses it back to text

🤔 Why Character-Level?

Real GPT uses subword tokens (pieces of words), but character-level is the simplest possible approach. Each character = 1 token. No complex tokenizer needed — just a dictionary lookup! It's perfect for learning.

Training Data

How do we teach the model to predict the next character? By showing it millions of examples: "given these characters, the next one is..."

🪟 Sliding Window

We slide a window of fixed size across the entire text. For each window position:

  • Input = first N characters (what the model sees)
  • Target = same characters shifted by 1 (what it should predict)

So the target is the input shifted right by one character. The model learns: "given input, predict target".

📖 Line-by-Line Breakdown

  • Line 1 — SEQ_LEN = 128: the model looks at 128 characters at a time. Longer = more context but slower training
  • Line 4 — Convert the ENTIRE text into numbers using our encode function. Result: one huge array of integers
  • Line 5 — Split: first 90% for training, last 10% for validation (to check if the model is learning, not memorizing)
  • Line 8 — from_tensor_slices: creates a TF dataset from the array (efficient data pipeline)
  • Line 9 — .window(SEQ_LEN + 1): create sliding windows. We need +1 because the target is shifted by 1
  • Line 10 — .flat_map: convert each window from a lazy dataset into an actual tensor
  • Line 11 — s[:-1] = input (first 128 chars), s[1:] = target (last 128 chars). The shift-by-one trick!
  • Line 12 — shuffle: randomize order (prevents the model from memorizing sequence order). batch: process 64 sequences at once (faster). prefetch: load next batch while GPU is busy

Embeddings

A single number (like 20 for 'H') can't capture meaning. Embeddings convert each token into a rich, learnable vector.

🧩 Token + Position Embeddings

Two separate embedding layers work together:

  • Token Embedding: Each character ID → a learned vector. The model learns that 'a' and 'e' are similar (both vowels) by making their vectors close together.
  • Position Embedding: Each position (0, 1, 2, ...) → a learned vector. This tells the model WHERE each character is. Without this, "abc" and "cab" would look identical!
  • We add them together: final = tok_emb + pos_emb. The result carries both "what" and "where" information.

🎮 Visualization — Embedding the word "cat"

Each character gets a token vector + position vector. The sum is what enters the transformer. (Showing 4 dimensions; real model uses 128)

embeddings in MiniGPT.__init__
1self.tok_emb = layers.Embedding(vocab_size, DIM) # character → vector
2self.pos_emb = layers.Embedding(SEQ_LEN, DIM) # position → vector

📖 Line-by-Line Breakdown

  • Line 1 — Embedding(vocab_size, DIM): creates a learnable lookup table of size 65 × 128. Each of the 65 characters gets its own 128-dimensional vector. These vectors are randomly initialized and learned during training.
  • Line 2 — Embedding(SEQ_LEN, DIM): creates a lookup table of size 128 × 128. Position 0 gets one vector, position 1 gets another, etc. This is how the model knows "this character is at position 5".
embeddings in MiniGPT.call()
1x = self.tok_emb(x) + self.pos_emb(tf.range(tf.shape(x)[1]))

📖 Explanation

self.tok_emb(x) looks up the vector for each character ID in the input. self.pos_emb(tf.range(...)) looks up vectors for positions 0, 1, 2, .... Adding them together gives each position a combined "what character + where it is" representation.

Causal Self-Attention

This is the ❤️ of GPT. Attention lets each character look back at all previous characters and decide which ones are most relevant.

🤔 The Intuition

Imagine you're reading: "The cat sat on the ___"

To predict the blank, you don't need every word equally. "cat" and "sat" are more useful than "the". Attention automatically learns to focus on the important parts.

Each character asks: "Of all the characters before me, which ones should I pay attention to?"

🔒 Why "Causal"?

In GPT, the model can only look backwards (at past characters), never forward. When predicting position 5, it can see positions 1-5, but NOT position 6+.

This is enforced by a causal mask — a triangular matrix that blocks attention to future tokens:

🎮 Causal Attention Mask for "Hello"

Each row shows what a character can see. 'H' can only see itself. 'e' can see 'H' and 'e'. And so on.

🧠 Multi-Head Attention

Instead of one attention operation, we run 4 in parallel (NUM_HEADS = 4). Each "head" can learn to focus on different things:

  • Head 1 might focus on grammar patterns
  • Head 2 might focus on character names
  • Head 3 might focus on nearby characters
  • Head 4 might focus on sentence structure

Their results are combined for a richer understanding. Each head operates on DIM/HEADS = 32 dimensions.

attention in MiniGPT
1# In __init__:
2self.attn = [layers.MultiHeadAttention(HEADS, DIM // HEADS) for _ in range(NLAYERS)]
3
4# In call(): for each layer i
5x = self.norm[2*i](x + self.attn[i](x, x, use_causal_mask=True))

📖 Line-by-Line Breakdown

  • Line 2 — Create a list of attention layers (one per transformer block). MultiHeadAttention(4, 32) = 4 heads, each with 32 dimensions
  • Line 5 — self.attn[i](x, x, use_causal_mask=True): The input x is used as both the query AND the value. This is "self-attention" — the sequence attends to itself. use_causal_mask=True is the magic flag that prevents looking at future tokens
  • Line 5 — x + ...: This is a residual connection. We add the attention output BACK to the original input
  • Line 5 — self.norm[...](): Layer Normalization keeps the values stable during training

🤔 Why Residual Connections?

When you stack many layers, information can get lost or distorted (the "vanishing gradient" problem). Residual connections create a shortcut — the original input is always added back. This means:

  • Even if a layer learns nothing useful, the information survives
  • Gradients can flow directly through the shortcut during training
  • It makes training deep networks much easier and faster

Feed-Forward Network

After attention gathers context, the feed-forward network processes it. It's just two Dense layers.

⚙️ What It Does

The feed-forward network (FFN) is applied to each position independently. Think of it as each character having its own little brain that thinks about the context gathered by attention.

  • Layer 1 — Expand: Project from 128 dimensions → 512 dimensions (with ReLU activation). This gives the model more "room to think".
  • Layer 2 — Compress: Project back from 512 → 128 dimensions.
  • Dropout: Randomly zeroes out 10% of values during training. This prevents overfitting (memorizing instead of learning patterns).
feed-forward in MiniGPT
1# In __init__:
2self.ffn = [keras.Sequential([layers.Dense(FF, activation="relu"),
3 layers.Dense(DIM), layers.Dropout(0.1)])
4 for _ in range(NLAYERS)]
5
6# In call(): for each layer i
7x = self.norm[2*i+1](x + self.ffn[i](x))

📖 Line-by-Line Breakdown

  • Lines 2-4 — Create a list of FFNs (one per transformer block). Each is a Sequential model: Dense(512, relu) → Dense(128) → Dropout(0.1). We create NLAYERS of these.
  • Line 7 — Same pattern as attention: x + self.ffn[i](x) = residual connection (add input back), then self.norm[...]() = layer normalization.

🤔 Why ReLU?

ReLU (Rectified Linear Unit) is the simplest activation function: relu(x) = max(0, x). It just zeroes out negative numbers. Without an activation function, stacked Dense layers would collapse into a single linear transformation — you'd get no benefit from depth. ReLU adds non-linearity, letting the network learn complex patterns.

The Full MiniGPT Model

Now let's see how all the pieces fit together in one class. This is the entire GPT architecture!

minigpt_model.py — The Complete Model
1class MiniGPT(keras.Model):
2 def __init__(self):
3 super().__init__()
4 self.tok_emb = layers.Embedding(vocab_size, DIM)
5 self.pos_emb = layers.Embedding(SEQ_LEN, DIM)
6
7 self.attn = [layers.MultiHeadAttention(HEADS, DIM // HEADS) for _ in range(NLAYERS)]
8 self.ffn = [keras.Sequential([layers.Dense(FF, activation="relu"),
9 layers.Dense(DIM), layers.Dropout(0.1)])
10 for _ in range(NLAYERS)]
11 self.norm = [layers.LayerNormalization() for _ in range(NLAYERS * 2)]
12 self.head = layers.Dense(vocab_size)
13
14 def call(self, x):
15 x = self.tok_emb(x) + self.pos_emb(tf.range(tf.shape(x)[1]))
16 for i in range(NLAYERS):
17 x = self.norm[2*i](x + self.attn[i](x, x, use_causal_mask=True))
18 x = self.norm[2*i+1](x + self.ffn[i](x))
19 return self.head(x)

📖 Complete Breakdown

  • Line 1 — We inherit from keras.Model so Keras handles training, saving, etc. for us
  • Lines 4-5 — Embeddings: token + position (already explained in Section 3)
  • Line 7 — Attention layers: one per transformer block. 4 heads × 32 dims each = 128 dims total
  • Lines 8-10 — Feed-forward networks: one per block. Dense(512, relu) → Dense(128) → Dropout
  • Line 11 — Layer norms: NLAYERS × 2 because each block needs TWO norms (one after attention, one after FFN)
  • Line 12 — Output head: Dense(vocab_size) projects from 128 dims to 65 (one score per character)
  • Line 15 — Combine embeddings: add token + position vectors
  • Lines 16-18 — The main loop: for each layer, do attention then FFN (both with residual + norm)
  • Line 19 — Output: convert final vectors to raw scores (logits) over all possible next characters

📐 Tensor Shapes Through the Model

Input: (64, 128) — batch of 64 sequences, each 128 chars
After embedding: (64, 128, 128) — each char is now a 128-dim vector
Through transformer blocks: (64, 128, 128) — shape stays the same!
After output head: (64, 128, 65) — 65 scores per position (one per possible char)

  • Line 3 — Loss function: SparseCategoricalCrossentropy measures how wrong the predictions are. "Sparse" because our targets are integers (not one-hot). from_logits=True because our model outputs raw scores, not probabilities
  • Line 4 — Adam optimizer with learning rate 3×10⁻⁴. Adam adapts the learning rate for each parameter individually, making training more stable. The learning rate controls how big each update step is
  • Line 5 — Track accuracy: what % of next-character predictions are correct
  • Lines 8-9 — model.fit(): the training loop! Iterate through training data for up to 20 epochs, checking validation loss after each epoch
  • Lines 10-11 — EarlyStopping: if validation loss doesn't improve for 3 consecutive epochs, stop training and restore the best weights. This prevents overfitting!

🔄 What Happens Each Epoch

  • Forward pass: input goes through the model → predictions come out
  • Loss calculation: compare predictions to actual targets → get a "wrongness" score
  • Backward pass (backpropagation): calculate how each weight contributed to the error
  • Update weights: Adam adjusts all weights slightly to reduce the loss
  • Repeat for every batch in the dataset = 1 epoch

Text Generation

After training, we can generate new text by feeding a prompt and letting the model predict one character at a time.

step5_generate.py
1def generate(prompt, length=500, temperature=0.8):
2 ids = encode(prompt)
3 for _ in range(length):
4 x = tf.constant([ids[-SEQ_LEN:]])
5 logits = model(x, training=False)[0, -1] / temperature
6 ids.append(int(tf.random.categorical([logits], 1)[0, 0]))
7 return decode(ids)

📖 Line-by-Line Breakdown

  • Line 2 — Encode the prompt string into token IDs
  • Line 3 — Loop: generate one character at a time, 500 times
  • Line 4 — Take the last SEQ_LEN (128) characters as context. Wrap in a list and convert to tensor (adds batch dimension)
  • Line 5 — Get model predictions. [0, -1] extracts the logits for the LAST position only (that's the next-character prediction). / temperature scales the confidence
  • Line 6 — tf.random.categorical samples a character from the probability distribution. Higher probability characters are more likely to be chosen, but there's randomness!
  • Line 7 — Decode all the IDs back to text and return

🤔 How Sampling Works

  • Low temperature (0.2): The highest-probability character dominates. Output is repetitive but coherent.
  • Temperature = 1.0: Original probabilities. Balanced creativity.
  • High temperature (2.0+): All characters become roughly equally likely. Output is random and creative but may be nonsensical.
  • Greedy (temp→0): Always pick the highest probability character. Deterministic but boring.

Summary — You Built a GPT!

🏗️ Everything We Built

Character Tokenizer Converts text ↔ numbers so the neural network can process it
Token Embedding Converts each character ID into a rich, learnable 128-dim vector
Position Embedding Adds position info so the model knows WHERE characters are
Causal Self-Attention Lets each character look back at relevant past characters
Feed-Forward Network Processes the attention output with Dense layers
Residual + LayerNorm Stabilizes training and preserves information through deep layers
Output Head Converts vectors into next-character probabilities

🌟 Key Takeaways

  • GPT is autoregressive — it predicts one token at a time, left to right
  • Everything is about next-token prediction — the entire model is trained to predict the next character
  • Attention is the magic sauce — it lets the model dynamically focus on relevant context
  • The causal mask prevents cheating — no looking at future tokens
  • Residual connections make deep networks trainable by preserving information
  • Temperature controls the creativity-coherence tradeoff at generation time
  • This same architecture (scaled up massively) powers ChatGPT, GPT-4, and other large language models!

🚀 Want Better Results?

  • Increase DIM (128 → 256+) for richer representations
  • Increase NLAYERS (4 → 8+) for deeper understanding
  • Increase HEADS (4 → 8+) for more diverse attention patterns
  • Train for more epochs on a GPU
  • Use a larger dataset
  • Switch from character-level to subword tokenization (like real GPT)
